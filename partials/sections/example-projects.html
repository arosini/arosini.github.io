The Progress Reporter: http://progressreporter.herokuapp.com

This Node/Express project pulls data from an SQL database, crunches it into some meaningful statistics, and then displays those statistics in tables and graphs. The SQL database is owned by a team of PhD students who are trying to build a program which uses NLP to help students learn their course materials. Students would go through the course materials (slides, video or audio usually) within the program, and answer questions every so often about what they just learned. When a student gets a question wrong, the program points them to some course material they could review before they attempt the question again. The goal of the project is to automate the process of providing reference materials after an incorrect response. The goal of my application was to visuialize the data they collected, as well as to see if the provided reference materials actually helps the students.
Stat Compiler for Dota 2: https://play.google.com/store/apps/details?id=edu.cmu.heinz.arosini.dotaapp

This is the Android app I wrote in graduate school. It wasn't a requirement to publish the app but I did anyway because I had some extra time. The app is for players of the game DotA (Defense of the Ancients) 2. It compiles the stats from their most recent games by sending requests to a web service we had to write as part of the same assignment. The web service was required to communicate with an existing API. I chose to work with the Steam (the owner of DotA 2) and DotA 2 APIs because DotA is without a doubt my favorite game ever. The application has 17 current installs and has had a total of 120 installs (only 1 of which is me, I promise).
This Website! https://github.com/arosini/arosini.github.io

I wrote this website using Middleman, HAML, SASS, and CoffeeScript. I chose Middleman because it contains all the front-end goodies that come with Rails, however was designed for static websites (such as this!) and doesn't have all the unecessary backend components. I like HAML/SASS/CoffeeScript over HTML/CSS/JS because I like being forced to write well indented code and don't see a need for tags. Also CoffeeScript stops me from shooting myself in the foot with the 'intricies' of JavaScript. I also used JQuery 1.9 and Bootstrap 3 because I find them very easy to use and full of great features.
